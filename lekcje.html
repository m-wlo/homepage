Tydzień IV

  • gitk (gui) + git lens (vscode)
  • <div> - nie ma znaczenia semantycznego, używamy gdy chcemy coś "opakować" (tylko kiedy nie istnieje odpowiedni semantyczny znacznik)
  • Własności:
    • Display:
      • block - generuje blok, obszar elementu, wymuszający break line przed i po elemencie, domyślnie zajmuje 100% szerokości
      • inline - nie wymusza break line i umieszcza on wszystko koło siebie, w pojednynczej linii, nie patrząc na ewentualny rozmiar obszaru
      • inline-block - połączenie block i inline, umożliwia tworzenie bloków o zadanej powierzchni, ale również pozwala na pozycjonowanie elementów obok siebie w linii
    • Szerokość/długość:
      • width i height - stała szerokość/długość elementu. Przykładowo, gdy zmieni się rozmiar okna to element dalej będzie miał stałą szerokość, która może wykraczać poza rozmiar okna, pojawi się overflow.
      • max-width i max-height - maksymalna szerokość/długość elementu. Element maksymalnie może mieć dany wymiar, przy zmianie wymiarów okna rozmiar jest aktualizowany, np. szerokosc okna spadła poniżej 600px i szerokość elementu jest zmieniejszana do aktualnego rozmiaru okna. Max-height rzadko używane.
      • min-width i min-height - minimalna szerokość/długość elementu. Analogicznie do własności powyżej.
    • Overflow:
      • hidden
        • - ukrywa scrollbar
      • visible
        • - pokazuje scrollbar
      • auto - automatycznie, umożliwia tworzenie scrollbarów dla samych elementów, nie dla całego okna
  • Jednostki:
    • px
    • %
    • vw i vh - viewport width i viewport height
  • Zapisy pełne/skrótowe:
    • Padding/margin:
      1. I. top, II. right, III.bottom, IV.left (jak zegarek)
      2. I. top/bottom II.left/right (wartości lustrzane)
    • Przykłady:
      1. padding-top: 3px;
        padding-right: 4px;
        padding-bottom: 5px;
        padding-left: 6px;
        jest równoważne
        padding: 3px 4px 5px 6px;
      2. padding-top: 3px;
        padding-right: 4px;
        padding-bottom: 3px;
        padding-left: 4px;
        jest równoważne
        padding: 3px 4px;
    • Border:
      • border-width: 3px;
      border-style: solid;
      border-color: #fe201a;
      jest równoważne
      border: 3px solid #fe201a
  • margin auto - nadaje wartość automatyczną dla marginesu (potrzebuje zdefiniowane rozmiary elementu
  • margin nie należy do elementu
  • box-sizing:
      border-box - przy tej własności rozmiar elementu zajmuje dokładnie taki obszar jak został mu nadany, padding oraz border dodawane są do 'wewnątrz'
      content-box - przy tej własności rozmiar elementu jest zależny od własności width/height oraz paddingu i borderu, które dodawane są na 'zewnątrz' elementu
  • Pozycjonowanie elementów przy własnoci - position: absolute jest względem nadrzędnego elementu, który ma własność - position: relative/absolute, jeżeli nie ma takiego nadrzędnego elementu pozycjonowanie odbywa się względem elementu html, czyli całej strony.
  • Pozostałe własności position
    • sticky - przydatny link
    • fixed
  • Float nie służy do budowania layoutów, dzięki własności float, inne elementy mogą opływać element z float
  • clear - dopasowuje wysokość elementu do do rozmiarów elementów z float, przyjmuje wartości: left (dopasowuje się tylko do lewego float), right (tylko do prawego), both (od obydwóch). Podobnie efekt daje dodanie overflow: hidden do elementu nadrzędnego
  • primitive values w js: String, Number, Bigint, boolean, null, undefined, symbol
  • immutability - nie zmieniamy wartości primitive values w js, zmieniamy tylko połączenia między nimi a zmiennymi
  • BEM (Block-Element_Modifier) - schemat nazewnictwa klas elementów block__element--modifier
    • Block - nadrzędny element
    • Element - część bloku (nie musi byćbezpośrednim dzieckiem)
    • Modifier - jeśli element występuje w kilku wariacjach tworzym dodatkową klasę
  • Propozycja workflow'u:
    1. semantyczny html
    2. dzielenie elementów na bloki
    3. dodanie klas elementów w html'u
  • Dodanie klasy czysto jsowej - dobra praktyka, pozwala na zachowanie funkcjonalności w JavaScripcie, a nazewnictwo klas stylingowych może się zmieniać.

    • flex: <flex-grow> <flex-shrink> <flex-basis>

Tydzień VI

  • Forms:
    • label
  • display: grid
    • Grid Line, Grid Track (Row/Column), Grid Cell, Grid Area
    • Numeracja rozpoczyna się od 1
    • grid-template-columns: <wartości po spacji> - tworzy kolumny o zadanym rozmiarze; działa tylko i wyłącznie bezpośrednio na swoje dzieci
    • grid-template-rows: <wartości po spacji> - tworzy wiersze o zadanej wysokości, w przeciwieństwie do kolumn opisuje jawnie określoną liczbę wierszy, nie "zapętla" kolejnych jak w przypadku kolumn
    • fr - fraction, jednostka rozmiaru wykorzystywana w grid, pojednynczy fr działa jak licznik ułamka, mianownikiem jest suma wszystkich fr, przykład: 1fr, 1fr, 2fr - pierwsze dwa elementu zajmują równo 1/4 rozmiaru (bo 1+1+2=4), a ostatni zajmuje 1/2 (2/4) rozmiaru. Fr mogą być mieszane z innymi wartościami, np. %, w takim wypadku rozmiar obliczany jest z dostępnego rozmiaru ograniczonego przez inne jednostki.
    • repeat(ilość, rozmiar) - funkcja css, powtarza dany element o danym rozmiarze zadaną ilość razy
    • column-gap, row-gap
    • minmax(min, max) - funkcja css, opisuje minimalny i maksymalny rozmiar elemntu
    • grid-template-area - opisuje "słownie" szablon layoutu (wiersze i kolumny), następnie z wykorzystaniem grid-area można przypisać poszczególne elemnty do obszarów zadeklarowanych w szablonie. Możliwe jest dodawanie obszarów nienazwanych, przez '.', należy wtedy pamiętać o rozmiarze dodatkowego, nienazwanego obszaru
    • grid-column-start, grid-column-end i grid-row-start, grid-row-end - własności opisujące "współrzędne" danego elementu na obszarze opisywanym przez grid-template-area. Dodatkowo, przy własnościach końcowych, można wykorzystywać z słowo kluczowe span, które opisuje ile kolumn/wierszy dany element ma zajmować, przykład: grid-column-end: span 3 - element będzie się rozciągał na 3 kolejne kolumny od początkowej.
    • Nazywanie obszarów grid - przykład: grid-template-columns: [pierwsza] 1fr [druga] 2fr [trzecia] 1fr
    • grid-area - może połączyć grid-column/row-start/end w jedno, schemat:
      grid-area: row-start / column-start / row-end / column-end
      Podobnie można skrócić pojedynczo row lub column poprzez własność grid-row lub grid-column, schemat:
      grid-column/row: start / end
  • Mobile first - pisanie stron w pierwszej kolejności na urządzenia mobilne, następnie za pomocą media queries dostosywywanie ich do urządzeń desktopowych
  • element.addEventListener(type, listener, [, options])
    • Mouse events:
      • click
      • mouseenter
      • mouseleave
      • mouseover
      • ...
    • Obiekt target - zawiera informacje o elemencie, którego event jest sprawdzany
